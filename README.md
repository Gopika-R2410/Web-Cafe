## ✨ Why This Project Exists

Most portfolio projects prove you can follow a tutorial. This one was built to prove something different — that I can **design and engineer an interaction**, not just style a page.

Web Café is a fully custom, from-scratch interface built around one question: *what does it feel like when every pixel responds to you?* The "coffee menu" is just the theme. The real subject is **motion, depth, and micro-interaction design**.

No UI kit. No template. No component library. Every animation, layout decision, and interaction pattern here was designed and hand-coded.

---

## 🎯 What Makes It Different

| Interaction | What's Actually Happening |
|---|---|
| 🖱️ **Custom Magnetic Cursor** | A cursor that morphs, trails, and reacts contextually as it moves across different UI zones |
| 🧲 **Proximity-Based Card Grid** | Cards scale, lift, and make room for each other in real time based on cursor distance — not just simple `:hover` |
| 🪟 **Glassmorphism System** | Layered blur, translucency, and depth used consistently across nav, cards, and modals |
| 🌗 **Animated Theme Switching** | A smooth, non-jarring dark/light mode transition — not a hard toggle |
| 🌀 **Parallax Hero with 3D Tilt** | The hero visual tilts toward your cursor using real-time pointer tracking |
| 🎬 **Scroll-Driven Reveals** | Staggered, physics-based entrance animations powered by GSAP + ScrollTrigger |
| 🔍 **Live Search & Filter** | Instant, animated filtering with zero page reloads |
| 📱 **Fully Responsive** | Interaction patterns adapt gracefully from desktop precision to mobile touch |

---

## 🛠️ Tech Stack

- **HTML5 / CSS3** — semantic structure, custom properties (design tokens), and layered glassmorphism
- **Vanilla JavaScript** — no framework overhead; all state, filtering, and DOM interaction hand-written
- **GSAP + ScrollTrigger** — timeline-based motion, easing curves, and scroll-synced animation
- **Deployed on GitHub Pages**

---

## 🧠 Design Principles Behind It

1. **Motion with purpose** — every animation communicates state (hover = interest, click = commitment, scroll = discovery), not decoration for its own sake.
2. **Consistency of depth** — glass, shadow, and blur values follow a single design system, not one-off effects.
3. **Cursor as a UI element** — treating the cursor itself as a designable surface, not an afterthought.
4. **Performance-conscious animation** — GPU-accelerated transforms, lazy-loaded imagery, and `prefers-reduced-motion` support baked in.
